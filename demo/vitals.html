Core Web Vitals: spine-first vs. full stylesheet

Same page, same content. Full CSS loads the whole stylesheet as a render-blocking <link>, so the first paint waits for it. Spine inlined ships the small layout skeleton in the HTML (no request, not render-blocking) and streams the paint complement.css in afterwards. Real networks aren't instant, so pick a simulated CSS latency — a service worker delays the CSS response in the browser, which works on static hosting (GitHub Pages) too.
